## 04. 구현 결과
✅ 완료 항목:
- 상태 태그의 앞뒤 공백, 연속 공백, zero-width 문자를 제거해 visually identical tag가 같은 값으로 처리되게 했다.
- 기존 Firestore mood 배열도 읽기 시 정규화해 선택 표시와 해제 동작이 일치하게 했다.
- 상태 저장/히스토리 저장 시 정규화된 mood 배열만 저장한다.
- 채팅 텍스트에 `http://` 또는 `https://` URL이 있으면 자동으로 새 창 링크가 된다.
- URL 뒤 마침표/쉼표/괄호 등 일부 문장부호는 링크 밖으로 분리한다.
- 고대비 모드에서도 링크가 밑줄과 대비 색으로 구분된다.

⚠️ 미완료 항목:
- 실기기에서 과거 저장된 상태 태그 조합 해제 시나리오 확인은 필요하다.
- URL 자동링크는 `http://`, `https://` 명시 URL만 대상으로 한다. `www.`만 있는 텍스트는 이번 범위에서 제외했다.

## 05. 특이점 / 결정사항
- 상태 태그 해제 실패 원인은 단순 `includes()` 비교가 공백/보이지 않는 문자/중복 값에 취약한 구조로 판단했다.
- 채팅 링크는 텍스트를 HTML로 주입하지 않고 React 노드로 분리 렌더링해 XSS 위험을 피했다.
