# Tether PRD

작성일: 2026-06-28  
기준 버전: v0.5.10  
대상: Tether PWA

## 01. 제품 개요

Tether는 연결된 커플 2명이 서로의 일상과 기록을 한 공간에서 공유하는 PWA 앱이다. 핵심은 채팅, 상태, 교환일기, 사진, 콘텐츠, 자료, 링크, 데이트 기록, 같이듣기를 작고 안정적인 커플 전용 공간으로 묶는 것이다.

## 02. 사용자

- 일반 사용자: 서로 연결된 커플 2명
- 관리자: `kuroicode@gmail.com`
- 접근성 기준 사용자: 고대비 모드, 다크 모드, 큰 글자 설정을 사용하는 사용자

## 03. 목표

- Google 로그인 기반으로 커플 연결과 복구를 안정화한다.
- 채팅, 사진, 파일, 일기, 상태, 콘텐츠를 커플 단위로 동기화한다.
- Log 페이지에서 릴리즈/핫픽스/기능개선/버그 리포트를 추적한다.
- 고대비/다크 모드에서 핵심 텍스트와 상태 정보가 명확히 보이게 한다.
- PC, Android PWA, iPhone PWA 환경에서 주요 기능을 유지한다.
- 음악 파일을 같이듣기 플레이리스트로 연결해 커플이 함께 듣는 경험을 제공한다.
- 자동 검증 게이트를 lint/build/Functions build/rules dry-run 기준으로 유지한다.

## 04. 주요 기능 범위

### 04-1. 인증과 연결

- Google 로그인만 허용
- 신규 커플 연결
- 기존 Google 계정 기반 복구
- 승인 대기 화면
- 재설치/재연결 시 FCM 토큰 재등록

### 04-2. 홈과 상태

- 커플 상태 카드
- 표정 아이콘, 상태 태그, 상태 메시지
- 상태 편집 후 확정 버튼을 눌렀을 때만 히스토리 저장
- 최근 활동에 내 활동과 상대 활동 표시

### 04-3. 채팅

- 텍스트, 이미지, 일반 파일, 음악 파일 전송
- 여러 파일 드래그앤드랍 및 붙여넣기 첨부
- 음악 파일은 채팅에서 직접 재생
- 상대가 읽기 전 읽음 전 표시, 읽은 뒤 숨김
- 채팅 사진은 사진첩으로 보낼 수 있음

### 04-4. 교환일기

- 일기 작성/열람
- 댓글 작성 및 댓글 알림
- 첫 화면에서 댓글 미리보기
- 읽음 처리는 카드 열람 기준

### 04-5. 자료실

- 채팅에서 업로드된 일반 파일과 문서를 모아 표시
- 음악 파일은 자료실에서 제외하고 같이듣기로 분리
- PDF, 문서, 압축파일 등은 열기/다운로드로 연결
- 파일 삭제와 검색 지원

### 04-6. 같이듣기

- 업로드된 음악 파일 전체 목록 표시
- 커플 각자 최대 3곡 선택
- 상대 곡은 수정 불가
- 내 화면에서 상대 곡 중 1곡 제외 가능
- 선택 후보를 상단 플레이어에서 전체 랜덤 반복 재생
- 플레이어 숨김 상태에서도 음악 재생 유지
- Home의 Player 버튼은 보이기/숨기기 토글

### 04-7. 콘텐츠, 링크공유, 데이트 레시피

- 콘텐츠: URL/이미지/상태/별점/리뷰
- 링크공유: 사이트명, URL, 요약 설명, 파일 첨부
- 데이트 레시피: 같이 먹은 음식의 날짜 로그

### 04-8. 사진첩

- 사진 업로드
- 코멘트 저장 및 카드 표시
- 채팅 사진을 사진첩으로 전송

### 04-9. 설정과 관리자

- 내 닉네임/프로필 사진 수정
- 상대 닉네임은 읽기 전용
- 알림 재등록 및 테스트
- 알림음 선택 및 미리듣기
- 관리자 페이지는 `kuroicode@gmail.com`만 접근

## 05. 비범위

- 네이티브 앱 수준의 시스템 푸시 제어
- 외부 URL 자동 썸네일 크롤링
- iOS 웹푸시 시스템 알림음 커스터마이징
- 전체 문서/파일 백오피스 자동 운영 도구

## 06. 성공 기준

- `npm run build` 통과
- Firebase Hosting 배포 완료
- Firestore rules 변경 시 dry-run 및 실제 배포 완료
- 자료실에는 비음악 파일, 같이듣기에는 음악 파일이 정확히 분리 표시
- 고대비/다크 모드에서 상태 태그, 메시지, 카드가 명확히 구분
- PC/Android/iPhone 주요 QA 시나리오 통과
- Google-only 가입 정책과 충돌하지 않는 live Firebase E2E 경로 유지

## 07. 리스크

- iOS PWA 푸시는 브라우저/OS 정책 영향이 크다.
- 브라우저 자동재생 정책상 음악 재생은 사용자 입력이 필요할 수 있다.
- 과거 채팅 파일은 `files` 인덱스 백필이 필요할 수 있다.
- 운영 데이터 직접 수정은 반드시 검증 보고서로 남겨야 한다.
- live Firebase E2E는 서비스 계정 또는 ADC 자격증명이 없으면 실행할 수 없다.
