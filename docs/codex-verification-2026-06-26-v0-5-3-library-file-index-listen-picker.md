## 04. 구현 결과
✅ 완료 항목:
- 자료실/같이듣기 데이터가 채팅 메시지 조회에만 묶이지 않도록 전용 파일 인덱스 추가
- 새 파일 업로드 시 파일 자료실 인덱스에 자동 저장
- 기존 메시지 기반 파일도 병합 표시해 과거 파일을 최대한 유지
- 같이듣기에서 업로드된 음악 전체 목록을 보고 직접 내 3곡에 추가/해제 가능
- `npm run build` 통과
- `firebase deploy --only firestore:rules --dry-run` 통과

⚠️ 미완료 항목:
- 과거 파일 중 현재 메시지 조회 범위 밖에 있는 항목은 클라이언트에서 자동 복원할 수 없음
- 필요 시 Admin SDK 기반 1회성 백필 스크립트로 과거 메시지 전체를 files 인덱스로 옮기는 작업 가능

## 05. 특이점 / 결정사항
- 자료가 삭제된 것은 아니며, v0.5.2에서 음악 파일을 자료실에서 숨기면서 같이듣기 선택 UI가 충분하지 않아 사라진 것처럼 보일 수 있었다.
- 새 구조에서는 자료실은 비음악 파일, 같이듣기는 음악 파일이라는 화면 분리는 유지하되, 파일 저장 인덱스는 공통으로 안정화했다.
