## 04. 구현 결과
✅ 완료 항목:
- 앱 창이 visible이면 서비스워커가 윈도우 시스템 알림을 띄우지 않음
- 앱 창이 visible이면 내부 토스트와 선택된 알림음만 재생
- 앱이 hidden/minimized일 때만 시스템 알림 표시
- 같은 `notificationId`가 foreground/SW 경로로 중복 도착해도 한 번만 처리
- `npm run build` 통과

⚠️ 미완료 항목:
- 실기기/실PC 포커스 전환 QA는 소장님 확인 필요

## 05. 특이점 / 결정사항
- 브라우저 FCM은 앱 창이 포커스를 잃은 상태에서 서비스워커 백그라운드 경로를 탈 수 있다.
- 따라서 `document.visibilityState`만 앱 내부에서 확인하는 것으로는 부족하고, 서비스워커의 `WindowClient.visibilityState`로 시스템 알림 표시 여부를 결정해야 한다.
