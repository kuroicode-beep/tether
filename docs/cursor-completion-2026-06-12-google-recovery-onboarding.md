# ✅ 완료 보고서 — Google 로그인 복구 안내 흐름 추가 (Cursor, 2026.06.12)

원본 작업지시문: 사용자 요청 — iOS 재설치 후 Google 로그인 시 코드 입력 화면이 나오는 문제 복구 가능하도록 수정  
기준 커밋: `ac2391a`  
작업 브랜치: `main`

## 01. 작업 요약

- 목표: Google 로그인 후 기존 커플 정보가 없을 때 코드 입력으로 바로 보내지 않고 복구 안내/재확인 흐름 제공
- 결과: 완료
- 소요 시간: 단기 핫픽스

## 02. 작업 로그

- Onboarding Google 준비 흐름에서 `createUserProfile` 대신 `createOrGetUserDoc` 사용
- Google 로그인 직후 기존 `coupleId`가 있으면 기존처럼 자동 복원
- Google 계정에 `coupleId`가 없으면 `recover` 단계로 이동
- 복구 화면에 현재 Google 이메일/UID 표시
- 기존 연결 재확인 버튼 추가
- 다른 Google 계정으로 로그인 버튼 추가
- 새 연결 진행 버튼 추가
- 새 연결 진행 시에만 새 invite code 생성
- Log 페이지에 복구 핫픽스 내역 추가

## 04. 구현 결과

### ✅ 완료 항목

- 기존 Google 계정에 연결 정보가 있으면 자동 복원 유지
- 연결 정보 없는 Google 계정이면 코드 입력 화면으로 바로 보내지 않음
- 사용자가 현재 로그인한 Google 이메일/UID를 확인 가능
- 복원 재시도와 다른 Google 계정 로그인, 새 연결 진행을 분리
- 새 연결 진행 전 데이터 병합 불가 가능성 안내

### ⚠️ 미완료 항목

- Google 연결 없이 익명 계정으로만 사용하던 데이터는 앱 삭제 후 자동 복원 불가
- 이 경우 기존 기기에서 Google 계정 연결을 먼저 해야 함

## 06. 특이점 / 결정사항

- 이미 연결된 Google 계정의 자동 복원은 유지하고, 새 Google UID로 잘못 새 커플 연결을 만들 위험을 줄이기 위해 복구 안내를 먼저 띄운다.
- 새 invite code는 사용자가 명시적으로 `새 연결 진행하기`를 눌렀을 때만 생성한다.
