## 04. 구현 결과
✅ 완료 항목:
- 파일 선택창에서 여러 파일 선택 가능
- 채팅창에 여러 파일 드래그앤드랍 가능
- 여러 파일은 한 번에 즉시 전송하지 않고 하나씩 미리보기/전송 확인
- 현재 파일 전송 후 다음 파일 미리보기 자동 표시
- 현재 파일 건너뛰기 가능
- 전체 취소 가능
- `Ctrl+V`로 클립보드 이미지/파일 첨부 가능
- 기존 이미지/음악/문서/zip 업로드 흐름 유지

⚠️ 미완료 항목:
- 다중 파일을 한 번에 일괄 전송하는 기능은 제외. 실수 전송 방지를 위해 순차 확인 방식으로 구현함.

## 05. 특이점 / 결정사항
- 파일 큐는 항상 "현재 미리보기 다음 파일들"만 저장한다.
- `ChatInput` 내부 paste는 이벤트 전파를 중단해 루트 paste와 중복 첨부되지 않게 했다.
- 루트 paste는 입력창 밖에서 붙여넣기한 파일도 잡는다.

## 07. 핸드오프 메모
- Cursor에게: `fileQueue`는 현재 preview를 제외한 남은 파일 목록이다. `slice(1)` 처리를 다시 넣으면 파일이 건너뛰어질 수 있으니 주의.
- Codex에게: rules 변경 없음. 기존 `sendFile`과 Storage path를 그대로 사용한다.
- 주의사항: 기존 사용자/환경 파일은 건드리지 않음.
